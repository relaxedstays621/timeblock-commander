# User Guide

Procedural operating layer.

Use this folder for:

- getting started
- local setup
- normal operating steps
- common commands
- task-oriented instructions

## Rule

Keep this folder procedural.

If a file starts becoming mostly about identity, role, or boundaries, move that material to `Purpose/`.

## Boilerplate

Typical starter files:

- `getting-started.md`
- `local-development.md`
- `common-commands.md`

Projects can rename these, but the folder should stay focused on operation rather than orientation.
